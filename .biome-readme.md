# Biome.js Setup

This project uses [Biome.js](https://biomejs.dev/) for linting and formatting.

## Available Scripts

- **`npm run lint`** - Check for linting issues in the `src` directory
- **`npm run lint:fix`** - Automatically fix linting issues
- **`npm run format`** - Check code formatting
- **`npm run format:write`** - Format code automatically
- **`npm run check`** - Run both linting and formatting checks
- **`npm run check:fix`** - Fix both linting and formatting issues automatically

## Quick Start

### Check all issues
```bash
npm run check
```

### Fix all issues automatically
```bash
npm run check:fix
```

## VS Code Integration

To integrate Biome with VS Code:

1. Install the [Biome VS Code extension](https://marketplace.visualstudio.com/items?itemName=biomejs.biome)
2. The extension will automatically use the `biome.json` configuration
3. Files will be formatted on save and linting errors will be shown inline

## Configuration

The Biome configuration is located in `biome.json`. Current settings:

- **Formatting**: 2-space indentation, single quotes, semicolons as needed
- **Line width**: 100 characters
- **Linting**: Recommended rules enabled with some custom rules
- **Import organization**: Automatically organizes imports

## What Biome Does

- **Linting**: Catches potential bugs and code quality issues
- **Formatting**: Ensures consistent code style
- **Import sorting**: Organizes imports automatically
- **Fast**: Written in Rust, much faster than ESLint + Prettier

## Notes

Biome has partial support for Vue files. It will check and format the `<script>` sections of `.vue` files, but not the `<template>` or `<style>` sections.
